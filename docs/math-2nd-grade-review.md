### 2.3 1학기 3단원: 덧셈과뺄셈

| 항목 | 내용 |
|------|------|
| **파일** | `AddSubGame.tsx` + `AddSubMenuPage.tsx` |
| **입력 모드** | numpad (전 유형) |
| **프레임워크** | GameShell 사용 |

#### 스펙 대비 구현 매핑

| 스펙 유형 | 구현 | 매핑 함수 | maxDigits | 부합 |
|-----------|------|-----------|:---------:|:----:|
| A: 두 자리 덧셈 (받아올림) | ✅ `two-add` | `genTwoAdd()` | 3 ✅ | ✅ |
| B: 두 자리 뺄셈 (받아내림) | ✅ `two-sub` | `genTwoSub()` | 2 ✅ | ✅ |
| C: 덧셈·뺄셈의 관계 | ✅ `relation` | `genRelation()` | 2 ✅ | ✅ |
| D: □에 알맞은 수 | ✅ `missing` | `genMissing()` | 2 ✅ | ✅ |
| E: 세 수의 계산 | ✅ `three` | `genThree()` | 3 ✅ | ✅ |
| F: 어떤 수 구하기 (서술형) | ✅ `story` | `genStory()` | 2 ✅ | ✅ |
| (추가) 받아올림·받아내림 세로셈 | ✅ `carry` | `genCarry()` | 3 | ⊕ |

#### 상세 분석

**✅ 잘 된 점:**
- 스펙의 6개 유형을 모두 구현하고, 추가로 세로셈 집중 유형(`carry`)을 넣어 **교과서 수준 이상**
- `genMissing()`이 4단계 난이도 제공: `□+a=b`, `a+□=b`, `a-□=b`, `□-a=b` (스펙 3단계보다 상세)
- `genStory()`가 5가지 문장제 템플릿 제공: 덧셈 상황, 뺄셈 상황, 어떤 수 구하기(+/-), 차이 구하기 (스펙 2가지보다 풍부)
- `genThree()`가 3가지 혼합: 덧+덧, 뺄-뺄, 덧-뺄 (스펙과 일치)
- `genRelation()`이 역연산 방향을 랜덤화 (`c-a=?` 또는 `c-b=?`)

**✅ 보완 완료 (2026-04-01):**
- **받아올림 보장**: `genTwoAdd()`가 일의 자리 합 ≥ 10을 보장하도록 수정 (`aOnes=2~9`, `bOnes=max(1,10-aOnes)~9`, `aTens/bTens=1~6`)
- **받아내림 보장**: `genTwoSub()`가 `bOnes > aOnes`를 보장하도록 수정 (일의 자리에서 뺄 수 없는 상황 항상 생성)
- **세로셈 UI 추가**: `genTwoAdd()`, `genTwoSub()`에 40% 확률로 `.addsub-vertical` 세로셈 레이아웃 출제 (가로/세로 혼합)
- **수 범위 조정**: `genStory()` type 3에서 `a, b = rnd(10, 44)` (합계 최대 88 ≤ 99, maxDigits:2 제약 준수)

**⚠️ 잔여 보완 사항:**
- **받아올림/받아내림 화살표 시각화**: SVG 화살표(십의 자리 올림 표시)는 미구현 → 향후 힌트 시스템과 연계

**교과서 부합 판정: ✅ 완전 부합 (초과 구현, 받아올림·내림 보장 완료)**

### 2.4 1학기 4단원: 길이재기

| 항목 | 내용 |
|------|------|
| **파일** | `LengthGame.tsx` (독립 UI, GameShell 미사용) |
| **입력 모드** | choices (전 유형) |
| **컴포넌트** | RulerSVG ✅ |

#### 스펙 대비 구현 매핑

| 스펙 유형 | 구현 | 매핑 | 부합 |
|-----------|------|------|:----:|
| A: 자 눈금 읽기 | ✅ `read0` + `readMid` | `genRead0()`, `genReadMid()` | ✅ |
| B: 길이 비교 | ✅ `compare` | `genCompare()` (3변형) | ✅ |
| C: 길이 어림 | ❌ 미구현 | — | ❌ |
| D: 임의 단위로 재기 | ❌ 미구현 | — | ❌ |
| E: 선분 길이 합·차 | ❌ 미구현 | — | ❌ |

#### 상세 분석

**✅ 잘 된 점:**
- `RulerSVG` 컴포넌트가 잘 구현되어 물체 위치 표시, 눈금 읽기 기능 동작
- `genRead0()`은 0cm에서 시작하는 기본형, `genReadMid()`는 중간 시작형 → 스펙 유형 A의 Level 1·2에 해당
- `genCompare()`가 3가지 변형 제공: 더 긴 것/더 짧은 것/차이 계산 → 스펙 유형 B와 일치
- 독립 UI로 구현하여 눈금자 레이아웃이 자연스러움

**❌ 미구현 유형:**
1. **길이 어림 (유형 C)**: 실물 크기 감각 문제 (4지선다). 스펙에는 연필 그림 + "대략 몇 cm?" 형태. **교과서 핵심 활동**이므로 보완 우선순위 높음
2. **임의 단위로 재기 (유형 D)**: 클립, 지우개 등 비표준 단위로 세기. 표준 단위(cm) 도입 전 단계이므로 교육적 중요도 중간
3. **선분 길이 합·차 (유형 E)**: 길이+연산 결합 문제. 3단원 덧셈·뺄셈과 연계하여 교육적 가치 높음

**⚠️ 입력 모드 차이:**
- 스펙: 유형 A·D·E → `numpad`, 유형 B·C → `choices`
- 구현: 전 유형 `choices` (4지선다) → 유형 A의 정확한 눈금 읽기 연습이 numpad보다 쉬워질 수 있음. 단, 터치 환경에서 어린이 편의성은 choices가 더 높으므로 현행 유지도 합리적

**교과서 부합 판정: ⚠️ 부분 부합 (핵심 3유형 구현, 2유형 미구현)**

## 부록: 코드 품질 관찰 사항

### GroupedItems 중복 정의
`MultiplicationIntroGame.tsx`와 `MultiplicationGame.tsx`에 `GroupedItems` 컴포넌트가 각각 독립 정의되어 있다. 스타일 클래스명도 다름(`.mult-group` vs `.mg-group-box`). → 공통 컴포넌트로 추출 권장.

### ArithmeticGame vs AddSubGame 관계
`ArithmeticGame.tsx`(연산 메뉴 → 받아올림/받아내림)와 `AddSubGame.tsx`(3단원 → 7유형)는 기능이 일부 겹침. 받아올림 덧셈은 두 곳에서 모두 제공. 정리 방안: ArithmeticGame은 "연산 연습" 용도(독립 캔버스 UI 유지), AddSubGame은 "단원 학습" 용도(GameShell 기반)로 역할 분리 유지.

### LengthGame의 독립 UI
`LengthGame.tsx`는 GameShell을 사용하지 않고 독립 UI로 구현됨. 눈금자 레이아웃 특성상 합리적 결정이나, 점수 관리·캔버스·힌트 등 GameShell의 공통 기능을 활용하지 못함. 향후 GameShell에 커스텀 display 영역을 확장하거나, LengthGame 자체에 힌트 로직을 별도 구현해야 함.

### ClockGame.tsx → TimeGame.tsx 전환 (Phase 7)
2학기 4단원 "시각과 시간" 게임이 `ClockGame.tsx`에서 `TimeGame.tsx`로 교체되었다. TimeGame은 기존 1유형(시계 읽기)에서 6유형(시간 계산·걸린 시간 포함)으로 확장된 버전이다. `ClockGame.tsx` 파일은 여전히 존재하나, App.tsx 라우트는 `TimeGame`을 import한다. 혼동 방지를 위해 ClockGame.tsx 삭제 또는 deprecated 표시 권장.

### Phase 7 신규 컴포넌트의 인라인 정의
GraphGame(PictureGraph, BarGraph), PatternGame(NumBox), FractionGame(FractionPie, FractionBar, Frac) 등의 시각화 컴포넌트가 각 게임 파일 내부에 인라인으로 정의되어 있다. 현재는 단일 게임에서만 사용하므로 문제 없으나, 향후 다른 게임에서 재사용 시 `components/` 디렉토리로 추출 필요.

> **최종 결론 (Phase 7 갱신)**: Phase 7에서 5개 단원(분류하기, 길이재기(m), 표와그래프, 규칙찾기)을 신규 구현하고, 시각과시간을 6유형으로 확장하며, 교과외 분수 단원까지 추가하여 **13개 단원 중 10개 완료, 2개 부분 구현, 1개 미구현**으로 전체 완성도가 대폭 상승했다. 유일한 미구현 단원인 **네자리수(2-1)**는 ThreeDigitGame 재사용으로 즉시 구현 가능하다. 부분 구현 단원 중 **곱셈구구**는 순차연습·곱셈표 추가, **시각과시간**은 시계 고르기·시계 맞추기·달력 3유형 추가가 필요하다. 힌트 기능은 **□ 채우기(역연산 힌트)**부터 시작하는 것이 효과 대비 구현 비용이 가장 낮다.
