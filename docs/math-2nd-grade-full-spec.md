## 2. 1학기 3단원: 덧셈과 뺄셈

### 학습 목표
- 두 자리 수끼리의 덧셈·뺄셈 (받아올림/받아내림 포함)
- 덧셈과 뺄셈의 관계 이해 (역연산)
- □에 알맞은 수 구하기
- 세 수의 덧셈·뺄셈 (앞에서부터 차례로)

### 현재 구현 상태 분석
ArithmeticGame.tsx에 받아올림 덧셈/받아내림 뺄셈이 있으나, 3단원의 전체 학습 목표(역연산, □ 구하기, 세 수 계산)는 미포함.

---

### 문제 유형 A: 두 자리 수 덧셈 (받아올림)
- **입력 모드**: `numpad` (maxDigits: 3)
- **예시 문제**:
  - `47 + 38 = ?` (세로셈 형태로 표시)
  - `56 + 27 = ?`
- **인터랙션**: 세로셈 UI에서 일의 자리→십의 자리 순서로 답 입력. 캔버스에 풀이 과정 필기 가능.
- **난이도 적합성**: 교과서 세로셈 형태 그대로 사용. 받아올림 1회만 발생 (두 자리+두 자리 범위). 기존 ArithmeticGame과 동일한 검증된 패턴.

### 문제 유형 B: 두 자리 수 뺄셈 (받아내림)
- **입력 모드**: `numpad` (maxDigits: 2)
- **예시 문제**:
  - `63 - 28 = ?`
  - `50 - 17 = ?` (십의 자리가 0이 되는 경우)
- **인터랙션**: 세로셈 UI. 받아내림 표시(십의 자리에서 빌려온 10 시각화).
- **난이도 적합성**: 1학기 수준에서 두 자리-두 자리까지만. 십의 자리가 0인 경우(예: 50-17)를 상위 난이도로 배치.

### 문제 유형 C: 덧셈·뺄셈의 관계
- **입력 모드**: `numpad` (maxDigits: 2)
- **예시 문제**:
  - `8 + 4 = 12 이므로, 12 - 8 = ?`
  - `덧셈식 15 + 7 = 22 를 뺄셈식으로 바꾸면? 22 - ? = 15` → 7 입력
- **인터랙션**: 덧셈식을 상단에 고정 표시하고, 관련 뺄셈식의 빈칸을 numpad로 채우기.
- **난이도 적합성**: 역연산 개념은 시각적 연결(화살표, 색상)로 직관 보조. 수의 범위를 한 자리→두 자리로 점진 확대.

### 문제 유형 D: □에 알맞은 수 구하기
- **입력 모드**: `numpad` (maxDigits: 2)
- **예시 문제**:
  - `□ + 15 = 42` → □ = ?
  - `73 - □ = 38` → □ = ?
- **인터랙션**: □를 노란색 깜빡이는 상자로 시각화. 답 입력 후 □ 자리에 숫자가 채워지는 애니메이션.
- **난이도 적합성**: 교과서에서 가장 중요한 유형. 빈칸 위치(앞/뒤)에 따라 난이도 조절. 뺄셈의 □가 앞에 오는 것이 가장 어려움.
- **난이도 파라미터**:
  - Level 1: `□ + a = b` (a, b ≤ 20, 받아올림 없음)
  - Level 2: `a + □ = b` (받아올림 포함)
  - Level 3: `□ - a = b` 또는 `a - □ = b` (받아내림 포함)

### 문제 유형 E: 세 수의 계산
- **입력 모드**: `numpad` (maxDigits: 3)
- **예시 문제**:
  - `12 + 25 + 8 = ?`
  - `50 - 14 - 9 = ?`
  - `34 + 18 - 27 = ?`
- **인터랙션**: 세 수 식을 가로로 표시. "앞에서부터 차례대로" 규칙 안내 텍스트 포함.
- **난이도 적합성**: 교과서에서 세 수의 덧셈/뺄셈/혼합을 다룸. 각 중간 결과가 100 이하가 되도록 수 범위 제한.

### 문제 유형 F: 어떤 수 구하기 (서술형)
- **입력 모드**: `numpad` (maxDigits: 2)
- **예시 문제**:
  - `"어떤 수에 17을 더했더니 45가 되었습니다. 어떤 수는?"` → 28
  - `"어떤 수에서 23을 빼면 19가 됩니다. 어떤 수는?"` → 42
- **인터랙션**: 짧은 문장을 큰 글씨로 표시. 핵심 숫자를 색상 강조. 캔버스에 식 세우기 유도.
- **난이도 적합성**: 교과서 서술형과 동일. 문장이 1~2줄로 짧고, 핵심 숫자를 강조하여 읽기 부담 최소화.

### 특수 UI 요소
- 세로셈 레이아웃 (기존 ArithmeticGame의 `.problem-vertical` 재사용)
- 받아올림/받아내림 화살표 SVG 시각화
- □ 빈칸: 노란색 테두리 + pulse 애니메이션
- 단계별 풀이 표시 (stepLines) 확장

## 3. 1학기 4단원: 길이 재기

### 학습 목표
- 여러 가지 단위로 길이 재기 (임의 단위)
- 1cm 알기, 자(ruler) 눈금 읽기
- 길이 어림하고 재기
- 길이 비교하기

---

### 문제 유형 A: 자 눈금 읽기
- **입력 모드**: `numpad` (maxDigits: 2)
- **예시 문제**:
  - [SVG 자 그림] 연필이 0cm부터 7cm까지 놓여 있다 → `"이 연필의 길이는 몇 cm인가요?"` → 7
  - [SVG 자 그림] 지우개가 2cm부터 8cm까지 → `"이 지우개의 길이는 몇 cm인가요?"` → 6
- **인터랙션**: SVG로 자(ruler)와 물건을 시각화. 자의 눈금은 1cm 간격으로 숫자 표시. 물건의 양 끝에 화살표.
- **난이도 적합성**: 교과서 자 눈금 읽기와 동일. 0에서 시작하는 것(Level 1) → 0이 아닌 곳에서 시작하는 것(Level 2).
- **특수 UI**: SVG ruler 컴포넌트 (눈금선 + 숫자 + 물건 그림)

### 문제 유형 B: 길이 비교
- **입력 모드**: `choices` (4지선다)
- **예시 문제**:
  - [두 개의 SVG 막대] `"더 긴 것은?"` → 선택지: 가 / 나
  - `"5cm와 8cm 중 더 긴 것은 몇 cm 더 긴가요?"` → 선택지: 1cm / 2cm / 3cm / 4cm
- **인터랙션**: 두 물건을 나란히 시각화. 터치 한 번으로 선택 완료.
- **난이도 적합성**: 시각적 비교는 저학년에 매우 적합. 숫자 비교로 확장하여 "몇 cm 더 긴가" 문제로 사고력 발전.

### 문제 유형 C: 길이 어림
- **입력 모드**: `choices` (4지선다)
- **예시 문제**:
  - [연필 그림] `"이 연필의 길이를 어림하면?"` → 선택지: 3cm / 7cm / 15cm / 30cm
  - `"교실 칠판의 길이는 대략?"` → 선택지: 5cm / 30cm / 3m / 50m
- **인터랙션**: 실물 크기 감각을 기르는 문제. 그림과 함께 4가지 보기 중 가장 적절한 것 선택.
- **난이도 적합성**: 일상 물건의 길이 감각은 교과서 핵심 활동. 어림은 정답 범위가 넓어 오답 부담이 적음.

### 문제 유형 D: 임의 단위로 재기
- **입력 모드**: `numpad` (maxDigits: 2)
- **예시 문제**:
  - [SVG: 클립 4개가 나란히 놓인 그림] `"이 색연필의 길이는 클립 몇 개인가요?"` → 4
  - [SVG: 지우개 3개 나란히] `"이 연필의 길이는 지우개 몇 개인가요?"` → 3
- **인터랙션**: 단위 물건이 반복 배치된 SVG 그림. 세어서 답 입력.
- **난이도 적합성**: 표준 단위(cm) 도입 전 단계. 시각적으로 세기만 하면 되므로 난이도 낮음.

### 문제 유형 E: 선분 길이 합·차
- **입력 모드**: `numpad` (maxDigits: 2)
- **예시 문제**:
  - `"가의 길이는 5cm, 나의 길이는 3cm입니다. 가와 나를 이으면 몇 cm인가요?"` → 8
  - `"12cm 리본에서 5cm를 잘랐습니다. 남은 길이는?"` → 7
- **인터랙션**: SVG로 선분·리본 시각화. 합치기/자르기 동작을 정적 그림으로 표현.
- **난이도 적합성**: 길이 개념과 연산을 결합. 1학기 3단원 덧셈·뺄셈과 연계.

### 특수 UI 요소
- **RulerSVG**: 0~15cm 범위의 자 눈금 SVG (1cm 간격, 5cm마다 긴 눈금)
- **ObjectOnRuler**: 자 위에 물건을 배치하는 SVG 합성 컴포넌트
- **BarCompare**: 두 막대를 나란히 비교하는 SVG

## 4. 1학기 5단원: 분류하기

### 학습 목표
- 기준을 정하여 분류하기
- 분류 기준의 객관성 이해
- 분류한 결과를 표로 정리하기
- 분류 결과 해석하기 (가장 많은 것, 적은 것 등)

---

### 문제 유형 A: 기준에 따라 분류하기
- **입력 모드**: `multiSelect`
- **예시 문제**:
  - [SVG: 빨간 삼각형, 파란 원, 빨간 원, 파란 사각형, 빨간 사각형, 파란 삼각형] `"빨간색인 것을 모두 고르세요"` → 빨간 삼각형, 빨간 원, 빨간 사각형
  - [SVG: 같은 도형들] `"삼각형을 모두 고르세요"` → 빨간 삼각형, 파란 삼각형
- **인터랙션**: 도형/물건 카드 6~8개를 격자로 배치. 각 카드를 터치하면 선택 토글(테두리 강조). 하단 "확인" 버튼으로 제출.
- **난이도 적합성**: multiSelect 모드 활용. 같은 도형 세트에서 기준만 바꾸면 다른 문제가 됨(색→모양→크기). 교과서의 "기준에 따라 결과가 달라진다" 개념 체험.

### 문제 유형 B: 분류 기준 맞추기
- **입력 모드**: `choices` (4지선다)
- **예시 문제**:
  - [SVG: 그룹 가={원, 원, 원}, 그룹 나={삼각형, 삼각형, 사각형, 사각형}] `"어떤 기준으로 분류한 것인가요?"` → 선택지: 색깔 / 모양 / 크기 / 개수
  - [SVG: 그룹 가={빨강들}, 그룹 나={파랑들}] `"분류 기준은?"` → 선택지: 색깔 / 모양 / 크기 / 무게
- **인터랙션**: 이미 분류된 결과를 보여주고, 그 기준을 유추하는 역문제. 터치 한 번으로 선택.
- **난이도 적합성**: 역발상 문제로 사고력 향상. 선택지가 4개여서 오답 부담 적음.

### 문제 유형 C: 분류 결과 세기
- **입력 모드**: `numpad` (maxDigits: 2)
- **예시 문제**:
  - [SVG: 과일 그림 여러 개] `"사과는 모두 몇 개인가요?"` → 5
  - [SVG: 동물 그림들] `"다리가 4개인 동물은 몇 마리?"` → 3
- **인터랙션**: 그림 속 특정 종류를 세어서 숫자 입력. 그림은 6~12개 배치.
- **난이도 적합성**: 세기 활동은 저학년 핵심. 대상이 8개 이하면 쉽고, 12개 이상이면 어려움.

### 문제 유형 D: 표 읽기
- **입력 모드**: `numpad` 또는 `choices`
- **예시 문제 (numpad)**:
  - [표: 좋아하는 과일 - 사과 5명, 바나나 3명, 포도 ?, 귤 2명 / 합계 14명] `"포도를 좋아하는 학생은 몇 명인가요?"` → 4
- **예시 문제 (choices)**:
  - [표: 탈것 - 버스 7, 지하철 5, 자전거 3, 택시 2] `"가장 많은 학생이 이용하는 탈것은?"` → 선택지: 버스 / 지하철 / 자전거 / 택시
- **인터랙션**: HTML 테이블을 깔끔하게 표시. 빈칸은 노란색 강조. 합계를 이용한 역추적 문제.
- **난이도 적합성**: 표 읽기는 교과서 핵심. 빈칸이 하나면 나머지로 역산 가능(덧셈·뺄셈 연계).

### 문제 유형 E: 올바른 분류 기준 판단
- **입력 모드**: `choices` (4지선다)
- **예시 문제**:
  - `"좋아하는 것과 싫어하는 것으로 분류하면 좋은 기준인가요?"` → 선택지: 좋은 기준이다 / 사람마다 다르므로 좋은 기준이 아니다
  - `"다음 중 누가 해도 같은 결과가 나오는 기준은?"` → 선택지: 맛있는 과일 / 빨간 과일 / 큰 과일 / 좋아하는 과일
- **인터랙션**: 기준의 객관성을 판단하는 개념 문제. 선택형으로 부담 없이 학습.
- **난이도 적합성**: 교과서에서 강조하는 "객관적 기준" 개념. 정답이 분명하여 저학년에 적합.

### 특수 UI 요소
- **ClassifyGrid**: 6~12개의 도형/물건을 격자로 배치하는 SVG 컴포넌트
- **GroupBox**: 분류된 그룹을 테두리로 묶어 표시하는 컴포넌트
- **DataTable**: 분류 결과 표 (빈칸 강조 가능)
- 기존 ShapeGame의 도형 SVG 재사용 가능

## 5. 1학기 6단원: 곱셈

### 학습 목표
- 여러 가지 방법으로 세기 (묶어 세기)
- "몇의 몇 배" 개념 이해
- 곱셈식 만들기 (×, = 기호)
- 덧셈식을 곱셈식으로 변환

---

### 문제 유형 A: 묶어 세기
- **입력 모드**: `numpad` (maxDigits: 2)
- **예시 문제**:
  - [SVG: 사과 3개씩 4묶음] `"사과는 모두 몇 개인가요?"` → 12
  - [SVG: 별 5개씩 3묶음] `"별은 모두 몇 개인가요?"` → 15
- **인터랙션**: 물건이 묶음별로 원(또는 점선 테두리)으로 구분되어 배치. 전체 개수를 세어 입력.
- **난이도 적합성**: 곱셈 개념 도입 단계. 시각적 묶음으로 "같은 수씩 반복"을 자연스럽게 체험.
- **난이도 파라미터**:
  - Level 1: 2~3개씩 2~4묶음 (답 ≤ 12)
  - Level 2: 4~5개씩 3~5묶음 (답 ≤ 25)

### 문제 유형 B: 몇의 몇 배
- **입력 모드**: `numpad` (maxDigits: 2)
- **예시 문제**:
  - `"3의 4배는 얼마인가요?"` → 12
  - [SVG: 구슬 2개 그림] `"2의 5배는?"` → 10
- **인터랙션**: "OO의 OO배" 텍스트를 크게 표시. 선택적으로 수직선(number line)에 뛰어세기 화살표 SVG 표시.
- **난이도 적합성**: "배" 개념은 곱셈의 핵심. 2~5의 배수 범위에서 시작하여 부담 없음.

### 문제 유형 C: 곱셈식 만들기
- **입력 모드**: `numpad` (maxDigits: 2)
- **예시 문제**:
  - [SVG: 접시 3개에 쿠키 4개씩] `"4 × 3 = ?"` → 12
  - `"2 + 2 + 2 + 2 + 2 를 곱셈식으로 나타내면 2 × ? = 10"` → 5
- **인터랙션**: 그림 → 곱셈식, 또는 덧셈식 → 곱셈식 변환. 빈칸 위치가 결과(곱) 또는 곱하는 수.
- **난이도 적합성**: 곱셈식의 의미를 반복 체험. 그림으로 곱셈 상황을 구체화.

### 문제 유형 D: 곱셈식 완성 (빈칸)
- **입력 모드**: `numpad` (maxDigits: 2)
- **예시 문제**:
  - `"□ × 3 = 15"` → □ = 5
  - `"4 × □ = 20"` → □ = 5
- **인터랙션**: □ 위치를 노란색으로 강조. numpad로 숫자 입력.
- **난이도 적합성**: 3단원 □ 구하기와 연계. 곱셈 범위를 2~5단으로 제한하여 저학년 수준 유지.

### 문제 유형 E: 덧셈식 ↔ 곱셈식 변환
- **입력 모드**: `choices` (4지선다)
- **예시 문제**:
  - `"3 + 3 + 3 + 3 을 곱셈식으로 바르게 나타낸 것은?"` → 선택지: 3×2 / 3×3 / 3×4 / 4×3
  - `"5 × 3 을 덧셈식으로 바르게 나타낸 것은?"` → 선택지: 5+3 / 3+3+3+3+3 / 5+5+5 / 3+5
- **인터랙션**: 변환 방향(덧셈→곱셈, 곱셈→덧셈)을 번갈아 출제. 선택형으로 터치 부담 최소화.
- **난이도 적합성**: 개념 이해 확인. 흔한 오류(3+3+3+3을 3×3으로 착각) 교정에 효과적.

### 특수 UI 요소
- **GroupedItems**: N개씩 M묶음의 물건 배치 SVG (원형 테두리로 묶음 표시)
- **NumberLine**: 수직선에 뛰어세기 화살표 표시 SVG
- 기존 □ 빈칸 pulse 애니메이션 재사용

## 6. 2학기 1단원: 네 자리 수

### 학습 목표
- 1000 알기, 네 자리 수 읽고 쓰기
- 각 자리의 값 (천의 자리, 백의 자리, 십의 자리, 일의 자리)
- 뛰어 세기 (1000씩, 100씩 등)
- 네 자리 수의 크기 비교

### 설계 방향
기존 ThreeDigitGame.tsx의 7가지 문제 유형을 4자리로 확장. 코드 구조를 거의 그대로 재사용 가능.

> **구현 참고**: ThreeDigitGame의 `generate()` 함수 내 수 범위(100~999)를 1000~9999로 변경하고, `maxDigits`를 4로 설정하면 기본 동작한다. 자릿값 색상 코드를 4색(천=빨강, 백=파랑, 십=초록, 일=노랑)으로 확장하고, 0이 포함된 수(예: 3056, 7040, 5002)를 난이도 상위 문제로 의도적 출제해야 한다. 특히 "가장 작은 네 자리 수 만들기"에서 0이 천의 자리에 올 수 없는 규칙이 핵심 학습 포인트이다.

---

### 문제 유형 A: 자릿값 알기
- **입력 모드**: `numpad` (maxDigits: 1)
- **예시 문제**:
  - `"4728에서 백의 자리 숫자는?"` → 7
  - `"3056에서 천의 자리 숫자는?"` → 3
- **인터랙션**: ThreeDigitGame 유형 ①과 동일. 자릿값 이름을 색상으로 강조.
- **난이도 적합성**: ThreeDigitGame의 검증된 패턴 확장. 0이 포함된 수(예: 3056)로 상위 난이도 조절.

### 문제 유형 B: 수 만들기 (자릿값 합성)
- **입력 모드**: `numpad` (maxDigits: 4)
- **예시 문제**:
  - `"1000이 3, 100이 5, 10이 2, 1이 8인 수는?"` → 3528
  - `"1000이 7, 10이 4인 수는?"` → 7040
- **인터랙션**: 자릿값별 개수를 색상 카드로 시각화. 0이 빠진 경우(십의 자리 0개 등) 주의.
- **난이도 적합성**: 세 자리 수에서 확장. 0이 포함된 수가 핵심 난이도 요소.

### 문제 유형 C: 뛰어 세기
- **입력 모드**: `numpad` (maxDigits: 4)
- **예시 문제**:
  - `"1000씩 뛰어 세기: 2000, 3000, 4000, ?"` → 5000
  - `"100씩 뛰어 세기: 4700, 4800, 4900, ?"` → 5000
- **인터랙션**: 수열을 가로로 나열, 마지막 칸을 빈칸으로. ThreeDigitGame 유형 ②와 동일 패턴.
- **난이도 적합성**: 자리올림이 발생하는 경계(예: 4900→5000)가 핵심 학습 포인트.

### 문제 유형 D: 크기 비교
- **입력 모드**: `numpad` (maxDigits: 4) 또는 `choices`
- **예시 문제 (numpad)**: `"3872와 3928 중 더 큰 수는?"` → 3928
- **예시 문제 (choices)**: `"빈칸에 알맞은 부등호는? 4521 ○ 4518"` → 선택지: > / < / =
- **인터랙션**: 두 수를 크게 나란히 표시. 자릿값별 비교 순서 시각화(천→백→십→일).
- **난이도 적합성**: ThreeDigitGame 유형 ④ 확장. 같은 자리까지 같고 아래 자리에서 차이나는 경우가 핵심.

### 문제 유형 E: 숫자 카드로 수 만들기
- **입력 모드**: `numpad` (maxDigits: 4)
- **예시 문제**:
  - `"숫자 카드 [2, 5, 0, 8]로 만들 수 있는 가장 큰 네 자리 수는?"` → 8520
  - `"숫자 카드 [1, 0, 3, 7]로 만들 수 있는 가장 작은 네 자리 수는?"` → 1037 (0이 천의 자리에 올 수 없음)
- **인터랙션**: 숫자 카드를 시각적으로 표시. "가장 큰/작은" 조건 강조.
- **난이도 적합성**: 0 포함 여부가 난이도 핵심(0이 천의 자리에 올 수 없음 → 판단력 요구).

### 특수 UI 요소
- ThreeDigitGame의 기존 UI 확장 (maxDigits: 4)
- 자릿값 색상 코드 (천=빨강, 백=파랑, 십=초록, 일=노랑)
- 숫자 카드 SVG (둥근 사각형 + 큰 숫자)

## 7. 2학기 2단원: 곱셈구구

### 학습 목표
- 2~9단 곱셈구구 암기 및 활용
- 1의 단, 0의 곱 이해
- 곱셈표에서 규칙 발견

### 설계 방향: 암기형 vs 풀이형 분리

곱셈구구는 **암기형**(속도·반복 중심)과 **풀이형**(사고력·응용 중심)을 명확히 분리한다.

| 구분 | 암기형 (Speed Drill) | 풀이형 (Problem Solving) |
|------|---------------------|------------------------|
| **목적** | 구구단 자동화·숙달 | 곱셈 개념 적용·사고력 |
| **프레임워크** | 독립 게임 루프 (GameShell 미사용) | GameShell 사용 |
| **입력** | numpad 전용 (빠른 응답) | numpad + choices 혼합 |
| **시간** | 타이머·콤보 (게임 요소) | 제한 없음 (차분히 풀기) |
| **피드백** | 즉시 전환 (0.8초) | 표준 정답 화면 |
| **파일** | `TimesTableDrill.tsx` (별도) | `TimesTableGame.tsx` (GameShell) |

> **진입 화면**: `TimesTablePage.tsx`에서 [⚡ 구구단 외우기] / [📝 구구단 연습] 두 버튼을 제공하여 분기한다.

---

### 🅰️ 암기형 (Speed Drill)

> GameShell을 사용하지 않고 **독립 게임 루프**로 구현. 연속 출제 + 타이머 + 콤보 시스템으로 반복 학습을 유도한다.

#### 문제 유형 A1: 순차 연습 (단별 순서대로)
- **입력 모드**: `numpad` (maxDigits: 2)
- **예시 문제**:
  - 3단 선택 시: `"3 × 1 = ?"` → 3, `"3 × 2 = ?"` → 6, … `"3 × 9 = ?"` → 27
- **인터랙션**: 선택한 단의 ×1부터 ×9까지 순서대로 출제. 한 바퀴 완주 시 축하 + 소요시간 표시.
- **난이도 적합성**: 구구단 암기의 가장 기본 형태. 순서대로 외우는 초기 단계에 적합.
- **교과서 순서**: 2단→5단→3단→6단→4단→8단→7단→9단 (단 선택 UI에 이 순서로 배치)

#### 문제 유형 A2: 랜덤 연습 (선택 단 랜덤)
- **입력 모드**: `numpad` (maxDigits: 2)
- **예시 문제**:
  - 3단 선택 시: `"3 × 7 = ?"` → 21, `"3 × 2 = ?"` → 6, `"3 × 9 = ?"` → 27 (랜덤 순서)
- **인터랙션**: 선택한 단 내에서 랜덤 출제. 9문제 한 세트 완료 시 결과 표시.
- **난이도 적합성**: 순차 암기 이후 확인 단계. 특정 곱에서 약한 부분 발견 가능.

#### 문제 유형 A3: 전체 랜덤 (종합 도전)
- **입력 모드**: `numpad` (maxDigits: 2)
- **예시 문제**:
  - `"7 × 4 = ?"` → 28, `"3 × 8 = ?"` → 24, `"9 × 6 = ?"` → 54 (2~9단 전체 랜덤)
- **인터랙션**: 2~9단 전체에서 랜덤 출제. 제한시간(60초) 내 최대 문제 풀기.
- **난이도 적합성**: 전체 구구단 숙달 확인. 타이머로 동기 부여. 6단 이상 곱이 빈번하게 출제되도록 가중치 설정.

#### 암기형 게임 메커닉

| 요소 | 설명 |
|------|------|
| **타이머** | 상단 프로그레스 바. 전체 랜덤: 60초 카운트다운. 순차/랜덤 연습: 무제한(소요시간만 기록) |
| **콤보** | 연속 정답 시 콤보 카운터 증가. 3콤보 이상: 불꽃 이모지 🔥. 5콤보 이상: ⚡ + 화면 흔들림 |
| **점수** | 정답 +10점, 콤보 보너스 +(콤보수×2)점, 오답 시 콤보 초기화 (감점 없음) |
| **오답 처리** | 정답을 빨간 글씨로 0.8초 표시 → 자동 다음 문제 (빠른 전환 유지) |
| **결과 화면** | 총 문제 수, 정답 수, 정답률(%), 최고 콤보, 소요 시간, 개인 최고 기록 갱신 여부 |
| **최고 기록** | localStorage에 단별 최고 기록(정답률·소요시간) 저장 |

#### 암기형 단 선택 UI

```
┌─────────────────────────────┐
│     ⚡ 구구단 외우기          │
├──────────┬──────────────────┤
│  [2단]⭐ │  [5단]⭐          │  ← 교과서 순서 배치
│  [3단]   │  [6단]            │  ⭐ = 완주 표시
│  [4단]   │  [8단]            │  (회색=미도전)
│  [7단]   │  [9단]            │
├──────────┴──────────────────┤
│  [🎯 전체 랜덤 도전]          │  ← 전체 단 합쳐서 60초 도전
│  [🔄 약한 구구단 연습]        │  ← 오답률 높은 곱만 집중 출제
└─────────────────────────────┘
```

- 2×4 격자로 단 버튼 배치 (교과서 학습 순서)
- 각 버튼: 단 숫자 + 완주 표시 (미도전: 회색, 완주: 금색 ⭐, 만점: 🌟)
- 단 버튼 터치 → 연습 모드 선택 (순차/랜덤 토글)
- 하단: [전체 랜덤 도전] + [약한 구구단 연습] (오답 데이터 기반)

### 🅱️ 풀이형 (Problem Solving)

> 기존 **GameShell** 프레임워크 사용. 여러 유형을 랜덤 혼합 출제하여 개념 적용력을 기른다.

#### 문제 유형 B1: 곱셈구구 빈칸 (역산)
- **입력 모드**: `numpad` (maxDigits: 1)
- **예시 문제**:
  - `"□ × 4 = 28"` → 7
  - `"6 × □ = 42"` → 7
  - `"□ × □ = 36 (같은 수끼리의 곱)"` → 6
- **인터랙션**: □ 위치를 노란색 pulse 애니메이션으로 강조. numpad로 숫자 입력.
- **난이도 적합성**: 곱셈구구를 활용한 역산(나눗셈 사고력 기초). □가 앞에 오는 것이 더 어려움.
- **난이도 파라미터**:
  - Level 1: `a × □ = b` (□가 뒤, 쉬운 단: 2, 5단)
  - Level 2: `□ × a = b` (□가 앞, 모든 단)
  - Level 3: `□ × □ = b` (같은 수의 곱, 제곱수만: 4, 9, 16, 25, 36, 49, 64, 81)

#### 문제 유형 B2: 곱셈표 빈칸 채우기
- **입력 모드**: `numpad` (maxDigits: 2)
- **예시 문제**:
  - [곱셈표: 행=2~5, 열=1~9, 3행 5열(3×5)이 비어 있음] `"빈칸에 알맞은 수는?"` → 15
  - [곱셈표: 7행 8열(7×8)이 비어 있음] `"? 에 알맞은 수는?"` → 56
- **인터랙션**: 곱셈표(행×열)를 HTML 테이블로 표시. 빈칸 한 칸을 노란색 강조. 행/열 헤더로 곱 유추 가능.
- **난이도 적합성**: 표 형태가 시각적 단서 제공. 교과서 곱셈표 활동과 일치.
- **난이도 파라미터**:
  - Level 1: 표 범위 2~5단, 빈칸 1개
  - Level 2: 표 범위 2~9단, 빈칸 1개
  - Level 3: 표 범위 2~9단, 빈칸 2~3개 (순차 입력)
- **특수 UI**: MultiplicationTable 컴포넌트 (행/열 헤더 색상 강조 + 빈칸 pulse 애니메이션)

#### 문제 유형 B3: 그림 보고 곱셈식 만들기
- **입력 모드**: `numpad` (maxDigits: 2)
- **예시 문제**:
  - [SVG: 봉지 4개에 사탕 6개씩] `"사탕은 모두 몇 개?"` → 24
  - [SVG: 꽃병 3개에 꽃 7송이씩] `"꽃은 모두 몇 송이?"` → 21
  - [SVG: 접시 5개에 딸기 4개씩] `"5 × 4 = ?"` → 20
- **인터랙션**: 1학기 6단원 묶어 세기의 확장판. 그림으로 곱셈 상황 인식 → 곱 계산.
- **난이도 적합성**: 시각적 맥락이 곱셈의 의미를 강화. 묶음(그룹) 수와 각 묶음의 개수를 세는 과정 포함.
- **특수 UI**: GroupedItems SVG 재사용 (1학기 6단원과 공유)

#### 문제 유형 B4: 곱셈구구 크기 비교
- **입력 모드**: `choices` (3지선다: >, <, =)
- **예시 문제**:
  - `"3 × 8 ○ 4 × 6"` → = (둘 다 24)
  - `"7 × 5 ○ 6 × 6"` → < (35 < 36)
  - `"9 × 3 ○ 4 × 7"` → < (27 < 28)
- **인터랙션**: 양쪽 곱셈식을 크게 나란히 배치. 가운데 ○ 자리에 부등호 선택. choices 배열: `['>', '<', '=']`.
- **난이도 적합성**: 양쪽 모두 계산해야 하므로 구구단 2회 활용. 같은 값(=)인 경우가 가장 어려움.
- **출제 전략**:
  - 같은 값 쌍 적극 활용: 교환법칙(3×8 = 8×3), 다른 식 동일 값(3×8 = 4×6 = 24)
  - 1 차이 쌍: 7×5=35 vs 6×6=36 (미세한 차이 판별)
  - 명확한 차이: 2×3=6 vs 9×8=72 (Level 1용)

#### 문제 유형 B5: 1의 곱과 0의 곱
- **입력 모드**: `numpad` (maxDigits: 2)
- **예시 문제**:
  - `"1 × 7 = ?"` → 7
  - `"5 × 1 = ?"` → 5
  - `"0 × 8 = ?"` → 0
  - `"6 × 0 = ?"` → 0
- **인터랙션**: "1을 곱하면 그대로!" / "0을 곱하면 언제나 0!" 규칙을 말풍선 힌트로 표시. 초반 3~5문제에만 힌트 표시 후 점차 제거.
- **난이도 적합성**: 교과서에서 별도로 다루는 특수 규칙. 개념 이해 후에는 쉬워지므로 초반에 집중 출제.

#### 문제 유형 B6: 곱셈구구 활용 서술형
- **입력 모드**: `numpad` (maxDigits: 2)
- **예시 문제**:
  - `"한 봉지에 사탕이 6개씩 들어 있습니다. 7봉지에는 사탕이 모두 몇 개?"` → 42
  - `"의자 한 줄에 8명이 앉을 수 있습니다. 4줄이면 모두 몇 명?"` → 32
  - `"색종이를 한 사람에게 3장씩 9명에게 나누어 주면 모두 몇 장?"` → 27
- **인터랙션**: 짧은 문장을 큰 글씨로 표시. 핵심 숫자 2개를 색상 강조(빨강+파랑). 캔버스에 곱셈식 세우기 유도.
- **난이도 적합성**: 교과서 서술형과 동일한 형태. 문장이 1~2줄로 짧음.
- **난이도 파라미터**:
  - Level 1: 2, 5단 범위 서술형
  - Level 2: 3, 4, 6단 범위 서술형
  - Level 3: 7, 8, 9단 범위 서술형

### 특수 UI 요소
- **MultiplicationTable**: N×M 곱셈표 HTML 테이블
  - 행/열 헤더 색상 강조, 빈칸 pulse, 정답 채우기 애니메이션
  - Props: `rows: number[], cols: number[], blanks: [r,c][]`
- **GroupedItems SVG**: 1학기 6단원 재사용 (묶음 물건 SVG)
- **DanSelector**: 단 선택 UI 컴포넌트
  - 2×4 격자, 완주 표시(⭐/🌟), 교과서 순서 정렬
  - Props: `onSelect: (dan: number) => void, records: Record<number, {best, completed}>`
- **TimerBar**: 제한시간 프로그레스 바 (암기형 전용)
  - 남은 시간 시각화, 10초 이하 시 빨간색 변환 + 깜빡임
- **ComboCounter**: 연속 정답 콤보 표시 (암기형 전용)
  - 콤보 수 + 이모지(🔥⚡), CSS scale bounce 애니메이션
- **DrillResult**: 결과 화면 (정답률·콤보·시간·최고 기록)

## 8. 2학기 3단원: 길이 재기 (m와 cm)

### 학습 목표
- 1m = 100cm 이해
- m와 cm를 함께 사용하여 길이 나타내기 (예: 1m 30cm)
- 길이의 합과 차 (같은 단위끼리)
- 길이 어림

---

### 문제 유형 A: 단위 변환
- **입력 모드**: `numpad` (maxDigits: 3)
- **예시 문제**:
  - `"150cm = ?m ?cm"` → 1m 50cm (두 단계: m→numpad, cm→numpad)
  - `"2m 30cm = ?cm"` → 230
- **인터랙션**: 단위 변환을 두 칸 입력으로 나누거나(m칸 + cm칸), 한 칸에 cm 총합 입력.
- **난이도 적합성**: 100cm = 1m 관계만 이해하면 됨. 1학기 길이 재기의 자연스러운 확장.
- **특수 UI**: 두 개의 입력 필드 (m □ cm □) 또는 단일 numpad

### 문제 유형 B: 길이의 합과 차
- **입력 모드**: `numpad` (maxDigits: 3)
- **예시 문제**:
  - `"3m 40cm + 2m 20cm = ?m ?cm"` → 5m 60cm
  - `"5m 70cm - 2m 30cm = ?m ?cm"` → 3m 40cm
- **인터랙션**: m끼리, cm끼리 따로 계산하는 과정을 시각화. 캔버스에 풀이 필기.
- **난이도 적합성**: "같은 단위끼리 계산" 원칙. cm의 합이 100 이상이 되는 경우(올림)는 고난이도.
- **난이도 파라미터**:
  - Level 1: cm 합/차에서 받아올림·받아내림 없음
  - Level 2: cm 합이 100 이상 (예: 70cm + 50cm = 120cm = 1m 20cm)

### 문제 유형 C: 길이 어림 (m 단위)
- **입력 모드**: `choices` (4지선다)
- **예시 문제**:
  - `"교실 문의 높이는 약?"` → 선택지: 50cm / 1m / 2m / 10m
  - `"아빠의 키는 약?"` → 선택지: 80cm / 1m 20cm / 1m 75cm / 3m
- **인터랙션**: 일상 물건의 길이 감각을 기르는 문제. 4지선다로 부담 없음.
- **난이도 적합성**: 실생활 연계. 오답이 명확하게 비현실적(10m 문 등)이어서 소거법 활용 가능.

### 특수 UI 요소
- **DualInput** (신규): m와 cm 두 칸 입력 UI
  - 레이아웃: `[   ] m  [   ] cm` 형태의 두 입력 필드
  - 활성 필드를 터치로 전환 (현재 활성 필드에 파란색 테두리 강조)
  - numpad는 하나만 공유, 활성 필드에 숫자 입력
  - [✓ 확인] 버튼 클릭 시 두 필드 값을 조합하여 정답 판정
  - GameShell 표준 모드가 아니므로 커스텀 입력 영역 또는 dualNumpad 모드 확장 필요
- 1학기 RulerSVG 확장 (m 단위 포함)
- **UnitBar** (신규): 100cm = 1m 관계를 시각화하는 색상 바 SVG

> **구현 우선순위**: cm→cm 변환 문제(예: "2m 30cm = ?cm")는 기존 numpad(maxDigits:3)로 즉시 구현 가능. m+cm 동시 입력이 필요한 유형은 DualInput 구현이 선행되어야 하므로, 초기 버전에서는 cm 단일 입력 유형만 먼저 구현하고, DualInput은 이후 확장한다.

## 9. 2학기 4단원: 시각과 시간

### 학습 목표
- 몇 시 몇 분 읽기 (5분 단위)
- 시각과 시간의 차이 이해
- 1시간 = 60분
- 달력 읽기 (요일, 날짜)

### ClockSVG 컴포넌트 활용 전략

이미 구현된 `ClockSVG` 컴포넌트(`src/components/ClockSVG.tsx`)는 **읽기 전용**(readOnly)과 **인터랙티브**(드래그) 두 모드를 모두 지원하므로, 이 단원의 핵심 UI로 활용한다.

#### ClockSVG Props 요약

```typescript
interface ClockSVGProps {
  hour: number;        // 0–11 (12시간제)
  minute: number;      // 0–59
  onChange?: (h: number, m: number) => void;  // 인터랙티브 모드용
  readOnly?: boolean;  // true=표시전용 (기본: false)
  size?: number;       // viewBox 크기 (기본: 280)
  showLabel?: boolean; // 디지털 시간 표시 (기본: true)
}
```

#### 문제 유형별 ClockSVG 사용법

| 유형 | 모드 | ClockSVG 코드 | 용도 |
|------|------|-------------|------|
| A. 시계 읽기 | readOnly | `<ClockSVG hour={3} minute={25} readOnly showLabel={false} />` | 시계 보고 시각 맞추기 |
| B. 시계 고르기 | readOnly ×4 | `<ClockSVG hour={h} minute={m} readOnly size={130} showLabel={false} />` | 4개 소형 시계 중 선택 |
| C. 시계 맞추기 | interactive | `<ClockSVG hour={12} minute={0} onChange={(h,m) => setUserTime(h,m)} />` | 목표 시각으로 바늘 조작 |
| D. 시간 계산 | readOnly | `<ClockSVG hour={startH} minute={startM} readOnly />` | 시작 시각 표시 |
| E. 걸린 시간 | readOnly ×2 | 두 시계 나란히 배치 | 시작·끝 시각 비교 |

> **핵심**: `showLabel={false}`로 디지털 시간 표시를 숨겨야 읽기 문제로서 의미가 있다. 정답 공개 시에만 `showLabel={true}`로 전환하여 피드백 제공.

> **GameShell 호환성**: 유형 A·D·E는 GameShell의 choices/numpad 모드로 처리 가능. 유형 B(시계 4개를 보기로 사용)와 유형 C(인터랙티브 시계)는 GameShell의 `display` 영역에 커스텀 UI를 배치하고, 별도 확인 로직이 필요하다.

---

### 문제 유형 A: 시계 읽기 (Clock → Time)
- **입력 모드**: `choices` (4지선다)
- **ClockSVG**: `<ClockSVG hour={h} minute={m} readOnly showLabel={false} />`
- **예시 문제**:
  - [시계: 3시 25분] `"지금 몇 시 몇 분인가요?"` → 선택지: 3시 20분 / **3시 25분** / 5시 15분 / 3시 30분
  - [시계: 7시 40분] `"시계가 가리키는 시각은?"` → 선택지: 7시 35분 / **7시 40분** / 8시 40분 / 7시 45분
  - [시계: 12시 00분] `"몇 시인가요?"` → 선택지: 6시 / **12시** / 11시 / 1시
- **인터랙션**: ClockSVG를 GameShell `display` 영역에 크게 표시 (시침=빨간색 #ff7e67, 분침=파란색 #5db8ff). choices 배열로 4지선다 표시.
- **오답 선택지 생성 전략** (교육적 오류 반영):
  - 시침·분침 혼동: 분침이 5를 가리킬 때 → "5시"로 오답 생성 (실제론 25분)
  - ±5분 인접 시각: 3시 25분 → 3시 20분, 3시 30분 오답
  - ±1시간 오차: 7시 40분 → 8시 40분 오답 (시침 위치 미세 차이 오인)
- **난이도 파라미터**:
  - Level 1: 정시만 (3시, 6시, 12시) — 분침 항상 12를 가리킴
  - Level 2: 30분, 15분, 45분 (반시, 쿼터) — 분침 6·3·9를 가리킴
  - Level 3: 5분 단위 자유 (3시 25분, 7시 40분, 10시 55분)

### 문제 유형 B: 시각 → 시계 고르기 (Time → Clock)
- **입력 모드**: `choices` (4지선다, 선택지가 시계 그림)
- **ClockSVG**: 소형 시계 4개를 `display`에 2×2 격자로 배치
  - `<ClockSVG hour={h} minute={m} readOnly size={130} showLabel={false} />`
  - 각 시계 아래에 라벨 "가", "나", "다", "라" 표시
- **예시 문제**:
  - `"4시 15분을 나타내는 시계는?"` → [가: 4:15 ✅] [나: 3:20] [다: 4:45] [라: 8:15]
  - `"9시 50분을 나타내는 시계는?"` → [가: 9:10] [나: 10:45] [다: 9:50 ✅] [라: 9:55]
- **인터랙션**: 상단에 목표 시각 텍스트를 크게 표시, 하단에 소형 시계 4개를 2×2 격자로 배치. choices 배열은 `['가', '나', '다', '라']`로 설정.
- **GameShell 연동**: `display`에 4개 시계를 라벨과 함께 렌더하고, choices의 텍스트를 "가/나/다/라"로 매핑. `answer`는 정답 시계의 인덱스.
- **난이도 적합성**: 유형 A의 역방향(인코딩) 연습. 유사한 시침 위치의 오답 시계 배치로 변별력 강화.

### 문제 유형 C: 시계 맞추기 (인터랙티브 — 신규 유형)
- **입력 모드**: 커스텀 (ClockSVG 인터랙티브 + 확인 버튼)
- **ClockSVG**: `<ClockSVG hour={12} minute={0} onChange={(h, m) => setUserTime(h, m)} />`
  - 초기값: 12시 0분 (바늘이 12에 모임)
  - 사용자가 "시" 손잡이, "분" 손잡이를 드래그하여 목표 시각 설정
- **예시 문제**:
  - `"시계를 3시 25분으로 맞추세요"` → 사용자가 바늘 드래그
  - `"아침 8시 30분에 학교에 갑니다. 시계를 맞추세요"` → 8시 30분 설정
  - `"오후 2시 45분을 시계에 나타내 보세요"` → 2시 45분 설정
- **인터랙션**:
  1. 목표 시각을 상단에 큰 텍스트로 표시 (예: `🎯 3시 25분`)
  2. ClockSVG 인터랙티브 모드 표시 → "⬆️ 손잡이를 잡아 돌려보세요!" 힌트 자동 표시
  3. 사용자가 시침·분침 드래그 (ClockSVG의 기존 포인터 이벤트 활용)
  4. 하단 [✓ 확인] 버튼 터치 → `onChange`의 최종 `(h, m)` 값과 목표 비교
  5. **정답 허용 범위**: 분침 ±2분 이내 (터치 정밀도 고려, 예: 목표 25분 → 23~27분 허용)
- **난이도 파라미터**:
  - Level 1: 정시만 (분침 고정, 시침만 이동하면 됨)
  - Level 2: 30분 단위 (시침이 두 숫자 사이로 이동하는 것 체험)
  - Level 3: 5분 단위 자유 설정
- **GameShell 호환**: GameShell의 표준 입력 모드가 아니므로 두 가지 방안 중 선택
  - 방안 1: GameShell `display`에 ClockSVG + [확인] 버튼을 배치, 확인 시 외부에서 정답 판정
  - 방안 2: ClockGame.tsx에서 별도 인터랙션 루프 구현 (GameShell 미사용)
  - **권장**: 방안 2 (인터랙티브 시계의 UX가 GameShell 레이아웃과 맞지 않을 수 있음)

### 문제 유형 D: 시간 계산 ("~시간 후/전")
- **입력 모드**: `numpad` (maxDigits: 2)
- **ClockSVG**: `<ClockSVG hour={startH} minute={startM} readOnly showLabel={false} />`
- **예시 문제**:
  - [시계: 2시] `"2시에서 3시간 후는 몇 시?"` → 5
  - [시계: 10시 30분] `"10시 30분에서 1시간 후는 몇 시 몇 분?"` → 11 (시만 입력, "OO시 30분"은 텍스트로 보조)
  - [시계: 4시 20분] `"4시 20분에서 2시간 전은?"` → 2 (시만 입력)
- **인터랙션**: 시작 시각을 ClockSVG readOnly로 표시, "OO시간 후/전" 조건을 큰 텍스트로 제시. 결과 시각의 "시" 부분만 numpad로 입력 (분은 변하지 않으므로).
- **난이도 적합성**: "시간 후/전" 계산은 교과서 핵심. 정시→정시부터 시작.
- **난이도 파라미터**:
  - Level 1: 정시에서 정수 시간 후 (2시 + 3시간 = 5시)
  - Level 2: 반시에서 정수 시간 후/전 (10시 30분 + 1시간 = 11시 30분)
  - Level 3: 12시간 경계 넘기 (11시 + 3시간 → "2시" — 시계가 한 바퀴 도는 개념)

### 문제 유형 E: 걸린 시간 구하기 (DualClock)
- **입력 모드**: `numpad` (maxDigits: 2)
- **ClockSVG**: 두 시계를 나란히 배치
  ```
  display 영역 레이아웃:
  ┌──────────────────────────────┐
  │   [시계1]   →   [시계2]      │
  │   시작 시각     끝 시각       │
  │                              │
  │  "몇 시간 동안 했나요?"       │
  └──────────────────────────────┘
  ClockSVG props: readOnly, size={130}, showLabel={false}
  ```
- **예시 문제**:
  - [시계1: 2시] → [시계2: 4시] `"영화는 몇 시간 동안 했나요?"` → 2
  - [시계1: 9시 30분] → [시계2: 10시 30분] `"수업은 몇 분 동안 했나요?"` → 60
  - [시계1: 1시] → [시계2: 4시] `"놀이공원에 몇 시간 있었나요?"` → 3
- **인터랙션**: 시작·끝 시각을 시계 2개로 시각화, 사이에 "→" 화살표 표시. 걸린 시간 입력.
- **난이도 적합성**: 시각(점)과 시간(구간)의 차이를 명확히 인식하게 하는 핵심 유형.
- **난이도 파라미터**:
  - Level 1: 정시 차이 (2시→4시 = 2시간)
  - Level 2: 같은 분, 시간만 차이 (9:30→10:30 = 1시간 = 60분)
  - Level 3: 3시간 이상 차이 (1시→4시 = 3시간)

### 문제 유형 F: 달력 읽기
- **입력 모드**: `choices` (4지선다) 또는 `numpad`
- **예시 문제 (choices)**:
  - [달력: 3월] `"3월 15일은 무슨 요일인가요?"` → 선택지: 월 / 화 / 수 / 목
  - [달력: 5월] `"어린이날(5월 5일)은 무슨 요일?"` → 선택지에서 선택
- **예시 문제 (numpad)**:
  - [달력: 4월] `"4월에서 수요일은 모두 몇 번 있나요?"` → 4 또는 5
  - [달력: 6월] `"6월은 모두 며칠까지 있나요?"` → 30
- **인터랙션**: CalendarGrid로 달력 표시. 문제에 해당하는 날짜/요일 강조. 공휴일·생일 등 친숙한 날짜 활용.
- **난이도 적합성**: 달력은 실생활 밀착. 요일 규칙성 발견으로 수학적 사고 연계.
- **특수 UI**: CalendarGrid 컴포넌트

### 특수 UI 요소
- **ClockSVG** (기존 구현 완료):
  - 아날로그 시계 (시침=빨간색 #ff7e67, 분침=파란색 #5db8ff)
  - 읽기 문제: `readOnly showLabel={false}` — 디지털 시간 숨김
  - 맞추기 문제: `onChange` 콜백으로 사용자 입력 수집
  - 보기용 소형: `size={130} showLabel={false}` — 4지선다 시계 보기
  - 정답 공개: `showLabel={true}`로 전환하여 디지털 시간 표시
- **DualClock** (신규): 두 시계를 "→" 화살표로 연결하여 나란히 배치하는 래퍼
  - Props: `startHour, startMinute, endHour, endMinute`
  - 내부: ClockSVG 2개 + 화살표 SVG + "시작"/"끝" 라벨
- **CalendarGrid** (신규): 월별 달력 HTML 테이블
  - Props: `year: number, month: number, highlights?: number[]`
  - 요일 헤더 (일~토, 일요일=빨강 #ff6b6b, 토요일=파랑 #5db8ff)
  - 날짜 셀 (하이라이트 날짜는 노란 배경 + 테두리)
  - 반응형: 모바일 화면에 맞게 셀 크기 조절
- **TimeLabel**: "OO시 OO분" 텍스트를 큰 글씨로 표시하는 컴포넌트 (목표 시각 표시용)

## 10. 2학기 5단원: 표와 그래프

### 학습 목표
- 자료를 표로 정리하기
- ○를 이용한 그래프(그림그래프) 그리기
- 표와 그래프 읽고 해석하기
- 표와 그래프의 장단점 이해

### 문제 유형 A: 표 읽기
- **입력 모드**: `numpad` (maxDigits: 2)
- **예시 문제**:
  - [표: 좋아하는 운동 - 축구 8, 야구 5, 농구 ?, 수영 3 / 합계 22] `"농구를 좋아하는 학생은 몇 명?"` → 6

- **난이도 적합성**: 표에서 정보 추출은 교과서 기본. 합계로 빈칸 역산은 덧셈·뺄셈 연계.

### 문제 유형 B: 그래프 읽기

  - [○ 그래프] `"가장 인기 있는 계절은?"` → 선택지: 봄 / 여름 / 가을 / 겨울
- **인터랙션**: ○(동그라미)를 세로로 쌓은 그래프를 SVG로 표시. ○ 개수=학생 수.
- **난이도 적합성**: 그래프는 시각적이어서 저학년에 매우 적합. ○ 세기는 기본 활동.
- **특수 UI**: PictographSVG (○ 그래프 컴포넌트)

### 문제 유형 C: 표 ↔ 그래프 변환

  - [○ 그래프 제시] `"그래프에서 바나나는 몇 명인지 표의 빈칸에 알맞은 수는?"` → (○ 개수)
- **인터랙션**: 표와 그래프를 동시에 표시. 한쪽의 정보로 다른 쪽 빈칸 채우기.
- **난이도 적합성**: 표↔그래프 대응은 자료 해석의 핵심. 두 표현 형식 간 변환 연습.

### 문제 유형 D: 자료 해석

- **예시 문제**:

  - [○ 그래프] `"가장 적은 것과 가장 많은 것의 차이는 몇 명?"` → 선택지: 2 / 3 / 4 / 5
- **인터랙션**: 자료를 읽고 해석하는 고차 사고 문제. 선택형으로 부담 감소.
- **난이도 적합성**: 교과서에서 "표와 그래프의 좋은 점" 비교 활동과 연계.

### 특수 UI 요소

- **PictographSVG** (신규): ○ 그래프 컴포넌트

  - **가로축**: 항목 이름 (예: 봄, 여름, 가을, 겨울)
  - **세로축**: 개수 눈금 (1~10, 1칸=1명)
  - **○ 스타일**: 파스텔 색상 원 (항목별 다른 색), r=14px, 간격=4px
  - **Props**: `data: { label: string; value: number; color: string }[]`, `maxValue?: number`
  - **크기 제한**: 최대 6개 항목, 각 항목 최대 10개 ○ (한 화면 표시 가능)
- **BarChartSVG**: 막대 그래프 (향후 확장용)

## 11. 2학기 6단원: 규칙 찾기

### 학습 목표
- 덧셈표에서 규칙 찾기
- 곱셈표에서 규칙 찾기
- 무늬에서 규칙 찾기
- 쌓은 모양에서 규칙 찾기
- 수 배열에서 규칙 찾기

---

### 문제 유형 A: 수 배열 규칙 (빈칸 채우기)
- **입력 모드**: `numpad` (maxDigits: 2)
- **예시 문제**:
  - `"2, 5, 8, 11, ?"` → 14 (규칙: +3)
  - `"30, 25, 20, 15, ?"` → 10 (규칙: -5)
- **인터랙션**: 수열을 가로로 나열, 마지막(또는 중간)을 빈칸으로. 규칙 힌트는 차이값 화살표로 시각화 가능.
- **난이도 적합성**: 등차수열(일정한 차이) 패턴만 사용. 차이가 1~10 범위.
- **난이도 파라미터**:
  - Level 1: 차이 1~3, 증가만
  - Level 2: 차이 5~10, 증가 또는 감소
  - Level 3: 빈칸이 중간에 위치

### 문제 유형 B: 덧셈표 규칙
- **입력 모드**: `numpad` (maxDigits: 2)
- **예시 문제**:
  - [덧셈표 일부 제시, 한 칸 비어 있음] `"빈칸에 알맞은 수는?"` → (행+열 값)
  - [덧셈표에서 대각선 강조] `"색칠된 칸의 규칙은?"` → choices로 전환
- **인터랙션**: 덧셈표(행 헤더 + 열 헤더 + 합 셀)를 HTML 테이블로. 빈칸 한 개 강조.
- **난이도 적합성**: 덧셈 숙달과 표 읽기를 동시에 연습.

### 문제 유형 C: 곱셈표 규칙
- **입력 모드**: `numpad` (maxDigits: 2)
- **예시 문제**:
  - [곱셈표 일부, 한 칸 비어 있음] `"빈칸에 알맞은 수는?"` → (행×열 값)
  - [곱셈표에서 같은 곱에 색칠] `"3 × 4와 같은 값을 가진 곳을 찾으면?"` → 4 × 3 = 12
- **인터랙션**: 곱셈표 빈칸 채우기. 교환법칙 발견 유도.
- **난이도 적합성**: 곱셈구구 복습과 규칙 발견을 동시에. 표 속 패턴은 시각적으로 발견 가능.

### 문제 유형 D: 무늬(패턴) 규칙
- **입력 모드**: `choices` (4지선다)
- **예시 문제**:
  - [SVG: ○△□○△□○△?] `"? 에 올 모양은?"` → 선택지: ○ / △ / □ / ☆
  - [SVG: 🔴🔵🔵🔴🔵🔵🔴?] `"다음에 올 색은?"` → 선택지: 🔴 / 🔵 / 🟢 / 🟡
- **인터랙션**: 도형이나 색 패턴을 SVG로 나열. 다음에 올 것을 선택.
- **난이도 적합성**: 패턴 인식은 저학년 수학의 핵심 역량. 반복 주기 2~3이면 쉽게 파악 가능.
- **난이도 파라미터**:
  - Level 1: 반복 주기 2 (AB AB AB)
  - Level 2: 반복 주기 3 (ABC ABC ABC)
  - Level 3: 반복 주기 4 또는 점진 변화 패턴

### 문제 유형 E: 쌓기나무 규칙
- **입력 모드**: `numpad` (maxDigits: 2)
- **예시 문제**:
  - [SVG: 1층=1개, 2층=3개, 3층=5개, 4층=?개] `"4층에 놓을 블록은 몇 개?"` → 7
  - [SVG: 계단 모양 1개→2개→3개→?개] `"다음 단계의 블록은?"` → 4
- **인터랙션**: IsoCubes(기존 ShapeGame 재사용) 또는 단순 블록 그림으로 패턴 시각화.
- **난이도 적합성**: 기존 ShapeGame의 IsoCubes 컴포넌트 활용 가능. 패턴이 시각적으로 명확.

### 특수 UI 요소
- **AdditionTable** (신규): 덧셈표 HTML 테이블
  - 행/열 헤더 색상 강조, 빈칸 pulse 애니메이션
  - Props: `rows: number[], cols: number[], blanks: [r,c][]`
  - 대각선/행/열 강조 기능 (규칙 발견 유도)
- **MultiplicationTable**: 7단원(곱셈구구)과 공유
- **PatternSequence** (신규): 도형/색 패턴 나열 SVG
  - 도형 세트: ○, △, □, ☆, ◇ (기존 ShapeGame SVG 재사용)
  - 색상 세트: 빨강 #ff6b6b, 파랑 #5db8ff, 노랑 #ffd93d, 초록 #6bcb77
  - 배치: 가로 나열 (최대 12개), 마지막(또는 중간)에 `?` 표시
  - Level 1 힌트: 반복 주기를 점선 그룹으로 구분 표시
  - Props: `items: {shape: string, color: string}[], blankIndex: number`
- **IsoCubes**: 기존 ShapeGame에서 재사용 (쌓기나무 패턴)

> **구현 참고**: 규칙 찾기는 다양한 유형(수열, 덧셈표, 곱셈표, 무늬, 쌓기나무)을 혼합 출제한다. GameShell의 `generate()` 함수에서 유형별 생성기를 랜덤 선택하는 패턴(MultiplicationIntroGame과 동일한 방식)을 사용한다. 수열 문제의 차이값(등차)은 문제 생성 시 1~10 범위 정수로 제한하고, 패턴 주기는 2~4로 제한한다.

## 12. 미니게임

### 12-1. 카드 짝맞추기 (Memory Match)

#### 게임 개요
- **목표**: 뒤집힌 카드 중 짝이 맞는 2장을 찾아 모두 뒤집기
- **학습 연계**: 곱셈구구, 덧셈·뺄셈, 단위 변환 등을 짝맞추기에 활용

#### 게임 모드
| 모드 | 카드 앞면 A | 카드 앞면 B | 예시 |
|------|------------|------------|------|
| 곱셈구구 | 곱셈식 | 결과 수 | `3×7` ↔ `21` |
| 덧셈뺄셈 | 덧셈/뺄셈식 | 결과 수 | `47+38` ↔ `85` |
| 단위변환 | cm 표현 | m+cm 표현 | `150cm` ↔ `1m 50cm` |
| 시계읽기 | 시계 그림 | 시각 텍스트 | [시계SVG] ↔ `3시 25분` |
| 도형 | 도형 이름 | 도형 그림 | `삼각형` ↔ [△ SVG] |

#### 인터랙션 설계
- **카드 배치**: 4×3 (12장=6쌍) 또는 4×4 (16장=8쌍)
- **터치 조작**: 카드 터치 → 뒤집기 애니메이션(CSS 3D transform)
- **짝 판정**: 2장 뒤집기 → 짝이면 사라짐(축소 애니메이션) / 아니면 다시 뒤집힘
- **클리어 조건**: 모든 짝 맞추기 완료
- **성과 지표**: 시도 횟수, 소요 시간

#### 난이도 파라미터
- Level 1: 6쌍 (4×3), 숫자 범위 작음
- Level 2: 8쌍 (4×4), 숫자 범위 확대
- Level 3: 8쌍, 유사한 값이 많아 혼동 유발 (예: 3×7=21, 3×8=24)

#### 특수 UI 요소
- **FlipCard**: CSS perspective + rotateY 뒤집기 애니메이션
- 카드 뒷면: 통일된 패턴(별, 물음표 등)
- 매치 성공: 파티클 효과 + 효과음
- 전체 클리어: 축하 화면 + 성적 표시

#### 프레임워크
- GameShell 사용하지 않음 (독립 게임)
- 별도 컴포넌트 구현 필요

### 12-2. 두더지잡기 (Whack-a-Mole)

#### 게임 개요
- **목표**: 올라오는 두더지 중 "정답 두더지"만 터치하여 점수 획득
- **학습 연계**: 빠른 암산 (곱셈구구, 덧셈, 크기비교 등)

#### 게임 모드
| 모드 | 화면 상단 문제 | 두더지에 표시 | 정답 조건 |
|------|--------------|-------------|----------|
| 곱셈구구 | `"3 × 7 = ?"` | 각 두더지에 숫자 (21, 15, 24, 18...) | 21인 두더지만 터치 |
| 크기비교 | `"가장 큰 수를 잡아라!"` | 각 두더지에 숫자 | 가장 큰 숫자 두더지 터치 |
| 홀짝 | `"짝수를 잡아라!"` | 각 두더지에 숫자 | 짝수 두더지만 터치 |
| 도형 | `"삼각형을 잡아라!"` | 각 두더지에 도형 그림 | 삼각형 두더지만 터치 |

#### 인터랙션 설계
- **구멍 배치**: 3×3 (9개 구멍) 격자
- **두더지 등장**: 랜덤 구멍에서 0.5~2초간 올라왔다가 내려감
- **터치 조작**: 두더지 터치 → 정답이면 ✅ + 점수 / 오답이면 ❌ + 감점
- **라운드**: 30초~60초 제한시간, 또는 문제 20개 출제
- **점수**: 정답 +10점, 오답 -5점 (감점은 선택적)

#### 난이도 파라미터
- Level 1: 두더지 등장 시간 2초, 동시 1~2마리
- Level 2: 등장 시간 1.5초, 동시 2~3마리
- Level 3: 등장 시간 1초, 동시 3~4마리, 함정 두더지(터치하면 감점) 추가

#### 특수 UI 요소
- **MoleHole**: 구멍 + 두더지 SVG (올라오기/내려가기 CSS 애니메이션)
- **두더지 캐릭터**: 귀여운 두더지 SVG (숫자/도형을 들고 있는 형태)
- **타이머**: 상단 프로그레스 바
- **점수판**: 실시간 점수 표시
- **효과**: 터치 시 먼지 파티클, 망치 애니메이션

#### 프레임워크
- GameShell 사용하지 않음 (독립 게임, 실시간 터치 기반)
- 별도 게임 루프 필요 (setInterval/requestAnimationFrame)

## 13. 입력 모드 확장 제안

현재 GameShell이 지원하는 3가지 모드(numpad, choices, multiSelect) 외에, 향후 확장이 필요할 수 있는 입력 모드를 제안합니다.

### 13-1. 순서 배열 (orderSelect) — 미구현, 향후 확장
- **용도**: 수의 크기 순서 배열, 시간 순서 배열
- **인터랙션**: 카드를 터치하여 순서대로 번호 부여 (1→2→3→4)
- **적용 단원**: 네 자리 수 크기비교, 시각과 시간
- **예시**: `"작은 수부터 순서대로 번호를 매기세요: 3854, 3548, 3845, 3458"` → 4,2,3,1

### 13-2. 드래그 매칭 (dragMatch) — 미구현, 향후 확장
- **용도**: 좌우 항목 연결 (곱셈식↔결과, 시계↔시각 등)
- **인터랙션**: 왼쪽 항목을 터치 → 오른쪽 항목을 터치하여 연결선 생성
- **적용 단원**: 곱셈구구, 시각과 시간, 단위 변환
- **예시**: 왼쪽[3×4, 5×2, 7×3] ↔ 오른쪽[12, 10, 21]
- **대안**: 현재 구현에서는 choices/multiSelect로 유사하게 대체 가능

### 13-3. 두 칸 입력 (dualNumpad) — 미구현, 향후 확장
- **용도**: m와 cm 동시 입력, 시와 분 동시 입력
- **인터랙션**: 두 개의 입력 필드 + 하나의 numpad. 터치로 활성 필드 전환.
- **적용 단원**: 길이 재기(m cm), 시각과 시간(시 분)
- **예시**: `"1m 50cm + 2m 30cm = [  ]m [  ]cm"` → 3, 80

> **구현 우선순위**: 현재 numpad/choices/multiSelect 3가지로 대부분의 문제 유형 커버 가능.
> 순서배열, 드래그매칭, 두칸입력은 해당 단원 구현 시점에 필요성 재평가.
